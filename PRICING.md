# Тарифная сетка VpnBot

Финансовая модель — продукт по **скорости**. Трафик-cap большой, после него мягкий throttle.

## Тарифы (актуальные)

| План | 1 мес ₽ | 1 мес ⭐ | Скорость | Сценарий | Soft-cap → throttle | Слоты |
|---|---|---|---|---|---|---|
| **База** | 200 ₽ | 180 ⭐ | 60 Mbps | 2 чел в 4K + телефоны | 500 GB → 5 Mbps | 2 AmneziaWG + 1 VLESS |
| **Макс** ⭐ | 500 ₽ | 450 ⭐ | 120 Mbps | Семья 3+, стриминг + торренты | 1 TB → 15 Mbps | 3 AmneziaWG + 5 VLESS |

### Логика «впритык» (естественный апсейл)
- **60 Mbps (База)** = 2× 4K (50 Mbps) + 10 Mbps запас на телефоны. Третий 4K не влезает → апсейл в Макс.
- **120 Mbps (Макс)** = 4× 4K + ещё запас. Хватает для дома и торрентов параллельно.

### Free trial для всех новых
**3 дня бесплатно** через Telegram-бот, без карты. Конверсия в платный 8-15%. Триал закрывается автоматически при первой покупке (warning показывается в UI: «оставшиеся бесплатные дни не переносятся»).

**7 дней для рефералов** — юзеры пришедшие по реферальной ссылке `/start ref_<id>` получают расширенный триал. См. [REFERRAL.md](./REFERRAL.md).

## Маркетинговая подача

Подаём по сценариям («2 чел в 4K + телефоны»), а не Mbps в лоб — клиенту проще понять value.

**Главный anchor**: годовая подписка −30%. За год эффективно 140 ₽/мес вместо 200₽ → юзер видит выгоду + retention для нас (кэш вперёд, не уходит каждый месяц).

## Источники истины

- **Backend**: `bot/services/plans.py` → `VPN_PLANS` dict (8 ключей: vpn_base / vpn_max + _3m/_6m/_12m суффиксы)
- **Frontend**: `webapp/src/components/PaymentSheet.tsx` → `STARS_PRICES` и `RUB_PRICES` таблицы (зеркало plans.py для UI)

При изменении цен — синхронить оба файла, иначе webhook не сматчит сумму с планом.

См. [PAYMENTS.md](./PAYMENTS.md) для подробностей по платёжным методам, recurring/cancel-флоу и архитектуре. См. [REFERRAL.md](./REFERRAL.md) для реферальной программы (7-day триал для рефералов + bank-redeem flow).
