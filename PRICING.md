# Тарифная сетка VpnBot

Финансовая модель — продукт по **скорости** (не по трафику). Трафик-cap большой, после него мягкий throttle.

## Финальная сетка v1

| План | Цена | Скорость | Сценарий | Soft-cap трафика → throttle |
|---|---|---|---|---|
| **Solo** | 350 ₽ / ≈$3.50 | **30 Mbps** | 1 человек 4K (впритык — второй уже не влезет) | 200 GB → 3 Mbps |
| **Family** ⭐ | 600 ₽ / ≈$6 | **50 Mbps** | 2 человека 4K параллельно (на пределе) | 600 GB → 8 Mbps |
| **Pro** | 1100 ₽ / ≈$11 | **100 Mbps** | 3+ человека 4K, торренты, стриминг | 1.5 TB → 25 Mbps |

### Логика «впритык» (естественный апсейл)
- **30 Mbps** = 1× 4K (25 Mbps) + 5 Mbps на телефон в фоне. Второй 4K — нет → апсейл в Family.
- **50 Mbps** = 2× 4K (50 Mbps) ровно. Третий девайс начинает страдать → апсейл в Pro.
- **100 Mbps** = практически безграничный для семьи + торренты.

### Атрибуты по планам

| Атрибут | Solo | Family | Pro |
|---|---|---|---|
| **Устройств одновременно** | 2 | 5 | 10 |

## Себестоимость и маржа

### Базис
Основная инфра — **Hetzner Cloud** (Helsinki / Falkenstein). DigitalOcean Frankfurt оставляем как backup-локацию.

| Hetzner CX22 | €5/мес | 1 vCPU, 4 GB, **20 TB трафика**, €1/TB овераж |
| Hetzner CCX13 | €13/мес | 2 dedicated vCPU, 8 GB — для Pro-нод (нужен запас по CPU на 100+ Mbps шифрования) |

### Расчёт на 1 ноду (~50 активных юзеров)

| План | Capacity на ноду | Себестоимость на юзера | Маржа на трафике | Маржа после OpEx (~5%) |
|---|---|---|---|---|
| Solo (30) | ~80 регистраций | ~$0.15 | 96% | ≈55% |
| Family (50) | ~50 регистраций | ~$0.40 | 93% | ≈53% |
| Pro (100) | ~25 регистраций (CCX13) | ~$0.85 | 92% | ≈47% |

OpEx учитывает: платёжный эквайер ~3.5% (CloudPayments / Stripe), Telegram-бот инфра ($5–10/мес фикс), поддержка, маркетинг с CAC ~$5–10 на конверсию.

**Маржа после всех расходов держится в коридоре 45–55%** на всех тарифах.

### Capacity-планирование

Пиковая шифровальная пропускная CX22 ≈ 80–100 Mbps на 1 vCPU. При peak-concurrency 20–30%:
- 80 Solo (30 Mbps) → 24 одновременных × 30 = 720 Mbps пик. CX22 не вытянет → **нужны 2 ноды на 80 Solo** или CCX13.
- 50 Family (50 Mbps) → 12 одновременных × 50 = 600 Mbps пик. CX22 не вытянет.
- **На практике:** Light/Standard = CX22 в Helsinki. Pro = CCX13 в Helsinki. AMS-FR/DO как fallback и SLA-region.

## Технические enforcement-механизмы

### Скорость = `tc HTB` per-port на eth0
Каждый план = отдельный набор inbound в Xray на отдельных портах:
- `reality-solo` — порты `8443–8452`, tc class `1:10` rate 30 mbit ceil 30 mbit
- `reality-family` — порты `9443–9452`, tc class `1:20` rate 50 mbit ceil 50 mbit
- `reality-pro` — порты `10443–10452`, tc class `1:30` rate 100 mbit ceil 100 mbit

Iptables CLASSIFY по `--sport` направляет трафик в нужный class.

### Soft-throttle = тоже отдельные inbound
- `reality-solo-throttled` — порты `11443+`, class `1:11` rate 3 mbit
- ...и т.д.

Когда юзер исчерпал квоту, агент через Xray API:
1. Удаляет UUID из «нормального» inbound
2. Добавляет в throttled-inbound
3. Subscription URL у юзера на следующем sync вернёт уже throttled-порты

### UUID-устройства
- 1 устройство = 1 UUID в Xray
- Solo: 2 UUID, Family: 5, Pro: 10
- Бот выдаёт нужное количество в subscription URL по плану клиента

## Проблема российских сервисов и split-tunnel

Российские сайты (Сбер, Госуслуги, vk, mail.ru, ozon, wildberries, gosuslugi, etc.) **блокируют foreign IP**. Решено через server-side routing:

```
iPhone → FR Reality → routing match *.ru → bot-server (151.243.113.31, RU-IP) → sberbank.ru
                  → routing match всё остальное → freedom outbound → интернет
```

**Реализация:**
- На bot-сервере поднят SS-2022 inbound (доступ ограничен firewall'ом для IP наших VPN-нод)
- На каждой VPN-ноде Xray routing.rules:
  ```json
  {"domain": ["geosite:category-ru"], "outboundTag": "ru-relay"},
  {"ip": ["geoip:ru"], "outboundTag": "ru-relay"}
  ```
- `geosite.dat` обновляется агентом из `github.com/v2fly/geoip` раз в сутки (cron)
- Прозрачно для клиента — любой VLESS-клиент работает

**Banking apps**: некоторые банки делают TLS-pinning или проверку через моб. оператора — split-tunnel сервера не помогает. В FAQ бота — инструкция «исключи app из VPN».

## Roadmap (см. отдельный файл)

Реализация тарифов требует доработки `vpnctl` агента и бота — описано в `AGENT_ROADMAP.md`.

Краткие приоритеты:
1. **P0** — UUID-CRUD через Xray API + sync с ботом (без этого нельзя enforce подписки)
2. **P1** — Multi-tier inbounds + tc HTB shaper (enforce скорости)
3. **P1.5** — Server-side RU-routing через bot-сервер (split-tunnel)
4. **P2** — Per-user soft-throttle при превышении квоты
5. **P3** — Capacity-aware распределение клиентов по нодам
